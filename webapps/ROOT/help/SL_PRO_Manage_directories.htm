To specify a new data directory

  1. Click Admin Settings, then click Data Directory.
  2. Click + in the Data Directory box.
  3. Enter a name for the new output data directory..
  4. Specify whether or not the data is coming directly from the instrument. <?? What does NO mean here?>.
  5. Specify the Transfer Protocol.  You can specify the path using one of three protocols: rsync daemon (rsy://), secure rsync (srs://), and samba (smb://).

    Note: The URI must be an absolute path, and cannot be a symbolic link.

    Example 1: rsy://pbi:Passw0rd@SMRTAnalysis.host.example.com:3333/pb0999/. (This uses rsync daemon mode.  module is defined on the destination server using the rsync daemon, and is an alias to a root directory under which all files will go.)

    Example 2: srs://pbi@host.example.com/pb09999/. (This uses rsync over ssh mode. No password or port numbers are allowed in the URI.)

    Example 3: smb://LABS;pbi:Passw0rd@host.example.com/pb09999/. (This uses samba mode, and can include an optional realm, such as lab name.) Note: If you are using samba, machines using RS Remote for instrument setup require the same samba access permissions as the instrument itself.
